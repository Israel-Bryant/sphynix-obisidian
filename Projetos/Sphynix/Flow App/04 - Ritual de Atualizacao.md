# Ritual de Atualizacao

## Quando Atualizar

Atualizar o Obsidian sempre que acontecer uma destas coisas:

- commit importante
- build enviada
- resposta da Apple ou Google
- decisao de produto
- mudanca de monetizacao
- alteracao de roadmap
- bug critico corrigido

## Passos

1. Atualizar [[00 - Flow App Dashboard]] com o estado resumido.
2. Atualizar [[01 - Progress Map]] se algum pilar mudou.
3. Registrar entrada em [[03 - Log de Progresso]].
4. Se for decisao permanente, registrar em [[../12 - Decisoes Irrevogaveis]].
5. Se mudar fonte de verdade, atualizar [[../13 - Fonte de Verdade]].
6. Se mudar contexto para IA, atualizar [[../99 - AI Context Pack]].

## Regra de Ouro

O dashboard mostra o presente.

O roadmap mostra o futuro.

O log mostra o que aconteceu.

As decisoes irrevogaveis mostram o que nao deve ser reaberto sem motivo forte.

## Campos Que Precisam Estar Sempre Certos

- branch ativa
- ultima build enviada
- status App Review
- risco Apple
- proxima acao
- decisao de monetizacao
- pilar mais atrasado
